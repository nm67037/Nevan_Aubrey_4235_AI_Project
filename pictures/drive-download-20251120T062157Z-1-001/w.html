The PARMCO System

Precision Automated Remote Motor Control & Observation. A closed-loop feedback system powered by Raspberry Pi and controlled via a custom Android Application.

View Setup How It Works
Hardware Architecture

Closed-Loop Feedback

The PARMCO system is more than just a motor driver. It utilizes a DC Motor coupled with an Optical Speed Sensor to create a feedback loop.

The Loop: The Raspberry Pi reads the sensor data to calculate the real-time RPM. It then adjusts the PWM (Pulse Width Modulation) signal sent to the H-Bridge driver to maintain consistent speed, even under varying loads.

  • ✓ Real-time RPM Sensing
  • ✓ Raspberry Pi 4 Processing Core
  • ✓ H-Bridge Motor Logic
App Capabilities

Dual Control Modes

The custom Android application allows for two distinct methods of operation, communicating wirelessly with the controller.

Manual Mode

Direct control over the motor's duty cycle. This mode bypasses the feedback loop, allowing the user to manually increment or decrement the power supplied to the motor.

Key Features:

  • • Direction Toggling (CW/CCW)
  • • Step-wise Speed Increment/Decrement
  • • Emergency Stop

Automatic Mode

Set a target, and the system handles the rest. Input a "Desired RPM" (e.g., 1200), and the Pi utilizes PID control logic to match the physical motor speed to your request.

Key Features:

  • • Exact RPM Targeting
  • • Self-Correcting Logic
  • • Real-time RPM Readout on App

The Build

Technical Specifications

The components that drive the PARMCO system.

  • Controller: Raspberry Pi 4 (GPIO Interface)
  • Actuator: DC Hobby Motor + Propeller
  • Feedback: IR Optical Speed Sensor (Encoder)
  • Driver: L293D H-Bridge IC
  • Client: Native Android Application
